# hermes-coach

AI cycling and endurance coach deployed on Kubernetes, powered by [hermes-agent](https://github.com/NousResearch/hermes-agent).

## What It Is

A multi-user Hermes Agent that acts as a personal endurance coach on Discord. It connects to [intervals.icu](https://intervals.icu) to retrieve each athlete's training data, then uses that data to give evidence-based coaching recommendations with full reasoning.

**Key design principle:** Coaching knowledge is decoupled from the agent core. Knowledge lives in versioned YAML files (`coach-brain/`), not in the system prompt. This preserves per-conversation prompt caching while keeping advice consistent and updatable.

## Architecture

```
Discord users
    │
    ▼
Hermes (Discord gateway + LLM + training tools)
    ├── intervals_icu    — fetch workouts, FTP, training load, wellness
    ├── weather          — Open-Meteo forecast (free, no key needed)
    ├── coaching         — retrieve structured Coach Brain knowledge
    ├── onboarding       — /start flow for connecting intervals.icu
    └── sandbox_client   — develop new tools autonomously (k8s Jobs)

Kubernetes (k3s single-node on Hetzner CS23)
    ├── hermes namespace       (main deployment)
    └── hermes-sandbox ns      (isolated job runner, network-isolated)
```

See [ARCHITECTURE.md](docs/ARCHITECTURE.md) for the full breakdown.

## Quick Start

```bash
# Clone and set up
git clone git@github.com:kvarnberg-labs/hermes-coach.git
cd hermes-coach

# Run tests
PYTHONPATH=plugins python -m pytest tests/ -q --import-mode=importlib

# Build Docker image
docker build -t ghcr.io/kvarnberg-labs/hermes-coach:dev .

# Deploy (Flux CD watches main branch)
git push origin main
```

See [DEVELOPMENT.md](docs/DEVELOPMENT.md) for local development and [DEPLOYMENT.md](docs/DEPLOYMENT.md) for production setup.

## Tools Reference

| Tool | Toolset | Purpose |
|------|---------|---------|
| `get_athlete_profile` | training | Fetch name, weight, timezone, resting HR |
| `get_sport_settings` | training | Fetch FTP, power zones, LTHR, W' |
| `get_recent_activities` | training | Fetch workouts over past N days |
| `get_wellness` | training | Fetch CTL, ATL, TSB, HRV, sleep, feel |
| `get_planned_events` | training | Fetch upcoming workouts and races |
| `get_power_curve` | training | Fetch peak power at standard durations |
| `get_weather` | weather | Current conditions + 48h forecast |
| `get_coaching_knowledge` | training | Retrieve coach-brain knowledge by topic |
| `coach_onboard` | training | Connect intervals.icu credentials |
| `develop_tool` | training | Develop new tools via sandbox |

## Secrets

Secrets are managed with SealedSecrets. See [DEPLOYMENT.md](docs/DEPLOYMENT.md) for sealing instructions.

```bash
# Seal a new secret
kubectl create secret generic hermes-discord-secret \
  --from-literal=DISCORD_BOT_TOKEN=<token> \
  --dry-run=client -o yaml | \
  kubeseal --cert ~/.hermes/coach-cluster-cert.pem \
  > apps/hermes/hermes-discord-sealed-secret.yaml
```
